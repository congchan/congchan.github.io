文本模糊匹配

发表于 2020-03-23 分类于 AI ， NLP 阅读次数： Valine：

在客服问答, 搜索等应用中, 把信息从文本中抽取出来后, 我们得到包含信息的文本片段和对应的信息(类型), 接下来就需要从已有的答案/数据库/网页中查找匹配.

大概总结了以下有如下问题

  1. 粒度不同: 对事物/人物的描述, 从空间上讲是从大到小, 比如描述地址, 公司, 职位一般是部分前缀, 如浙江杭州和杭州, 只有粗粒度信息无法保证信息精准, 只有细粒度的信息无法保证没有歧义.
  2. 顺序不同: 对事件的描述, 可以是时空顺序, 如1994年洛杉矶大地震和洛杉矶大地震(1994年), 也可以是因果顺序.
  3. 描述文本不同: 可能是多种描述方式, 更可能是错字, 如苹果和平果
  4. 其他混合类型: 如上海浦东和浦东新区

词典 -> 分词

词典的构建是核心, 只有构建完善的词典和词性, 才能完整地分词. 但构建词典往往涉及大量的规则. 有时候可能根据任务的特性衍生出多个词典, 还需要规则先判断选择哪种词典.

比如, 公司名称大概包含三部分

  1. 地理位置
  2. 关键词: 如重金属, 金融
  3. 公司属性: 如有限责任公司, 玩具厂、美容店
  4. 分支: 如xx分公司, xx支行

每一部分都需要构建词典, 比如地理位置词典, 行业词典…
